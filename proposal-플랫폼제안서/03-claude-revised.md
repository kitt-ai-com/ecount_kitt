## 3. Phase 1: 크롬 확장 프로그램 (즉시 도입)

### 3.1 개요
이카운트 ERP 화면에서 바로 사용할 수 있는 Chrome Extension입니다.
기존 업무 흐름을 변경하지 않고 자동화를 적용합니다.

### 3.2 핵심 기능 + 승인/검수 프로세스

#### 주문·물류 자동화
| 기능 | 설명 | 절감 | 검수 방식 |
|------|------|------|-----------|
| 카톡 주문 수집·입력 | 카톡 → AI 파싱 → 판매입력 변환 | 30분→2분 | **미리보기 확인 후 수동 승인** |
| 엑셀 판매입력 업로드 | 엑셀 → 일괄 등록 | 45분→3분 | **미리보기 테이블 → 건별 체크 → 실행** |
| 물류센터 양식 변환 | 이카운트 → 센터별 포맷 → 메일 | 20분→1분 | 변환 결과 미리보기 → 메일 발송 전 확인 |

#### 견적·영업 자동화
| 기능 | 설명 | 절감 | 검수 방식 |
|------|------|------|-----------|
| 거래처별 단가 추천 | 과거 패턴 학습 → AI 추천 | 신규 | 추천값 표시 → 사용자가 최종 선택 |
| 자동 견적서 생성 | 견적서 제작 + 이메일 | 20분→3분 | 견적서 미리보기 → 승인 후 발송 |
| 미전환 견적 알림 | 7/14일 무응답 자동 알림 | 신규 | 알림 목록 확인 → 선택적 발송 |
| 가견적 (박스 추천) | 치수 → 최적 박스 + 견적 | 신규 | 추천 결과 검토 → 수정 가능 |

#### 결제·매입·재무·인사
| 기능 | 설명 | 절감 | 검수 방식 |
|------|------|------|-----------|
| 세금계산서 매칭 | PG ↔ 이카운트 자동 비교 | 40분→5분 | 불일치 건만 표시 → 수동 확인 |
| 인보이스 OCR | PDF → AI OCR → 구매입력 | 15분→2분 | **OCR 결과 편집 가능 → 확인 후 등록** |
| 채권현황 자동 갱신 | 매일 자동 | 1시간→자동 | 갱신 결과 로그 → 이상치 알림 |
| 급여 원클릭 | 급여대장 → 이체서+명세서+메일 | 2시간→5분 | **금액 검증 → 관리자 최종 승인 필수** |
| 영수증 OCR | 영수증 → AI 분류 → 비용 리스트 | 신규 | 분류 결과 수정 가능 → 확인 후 저장 |

### 3.3 자동화 안전 원칙

> **핵심 원칙: AI는 제안하고, 사람이 승인한다**

| 업무 위험도 | 자동화 수준 | 예시 |
|------------|------------|------|
| **높음** (금전/인사) | AI 초안 → **반드시 수동 승인** | 급여 이체, 판매전표 등록 |
| **중간** (데이터 입력) | AI 변환 → **미리보기 확인 후 실행** | 엑셀 업로드, OCR 결과 |
| **낮음** (조회/알림) | 자동 실행 → **결과 알림** | 채권 갱신, 견적 알림 |

**실패 시 복구 절차:**
1. 모든 자동 등록 건에 고유 배치 ID 부여
2. 실패/오류 시 배치 단위 롤백 기능 제공
3. 이카운트 API 오류 → 재시도(3회) → 실패 시 수동 처리 안내
4. 모든 처리 이력은 로컬 스토리지에 90일 보관

**데이터 외부 전송 범위:**

| 전송 대상 | 전송 데이터 | 목적 | 비고 |
|-----------|------------|------|------|
| 이카운트 API | 판매/구매/재고 전표 데이터 | ERP CRUD | 이카운트 공식 API, 데이터 이카운트 서버에 저장 |
| Claude API | OCR 대상 문서 (PDF/이미지) | AI 파싱 | **Anthropic 정책: 입력 데이터 학습 미사용** |
| Claude API | 카톡 주문 텍스트 | AI 파싱 | 동일 |

**전송하지 않는 데이터:**
- 이카운트 로그인 비밀번호 (API 인증키만 사용)
- 급여 상세 내역 (로컬에서 엑셀 파싱, AI API 미전송)
- API 키 (chrome.storage.local에 암호화 저장, 외부 미전송)

> **"중간 서버 없음" 정정:** Phase 1은 브라우저에서 이카운트/Claude API를 직접 호출하므로 자체 백엔드 서버가 없습니다. 단, OCR/파싱 시 문서 데이터가 Claude API(Anthropic 서버)로 전송됩니다. Phase 2부터는 자체 백엔드 서버를 사용합니다.

### 3.5 AI 문서 처리 품질 관리

| 항목 | 기준 |
|------|------|
| OCR 정확도 목표 | 95% 이상 (필드 단위) |
| 검증 방법 | 고객 샘플 문서 30건으로 사전 테스트 |
| 사람 확인 UX | 모든 OCR 결과는 편집 가능한 테이블로 표시, 수정 후 등록 |
| 오류 케이스 | 인식 불가 필드는 빈칸 + 빨간색 표시 → 수동 입력 유도 |
| 미지원 서식 | 파싱 실패 시 "수동 입력 모드"로 전환, 원본 파일 표시 |

> **선행 조건:** 킥오프 시 CNC코리아의 실제 PG사 엑셀, 인보이스 PDF, 카톡 주문 샘플을 30건 이상 수집하여 파싱 정확도를 사전 검증합니다. 95% 미달 시 해당 워크플로우 범위를 조정합니다.

### 3.6 변화관리 및 배포 계획

## 4. Phase 2: AI 통합 대시보드 플랫폼

### 4.1 개요
이카운트 ERP 데이터를 실시간 수집·분석하는 독립 웹 플랫폼입니다.
Phase 1의 자동화에 더해 전사 데이터 시각화·분석·보고서 자동 생성을 제공합니다.

### 4.2 핵심 기능

#### 실시간 대시보드
- KPI 카드: 오늘 매출, 주문 수, 미수금, 재고 경고
- 매출 추이 차트: 일/주/월/연 매출 비교
- 채널별 매출 비중: 스마트스토어 / 자사몰 / 카톡 B2B
- 거래처 활동성: 활성 / 주의 / 휴면 / 이탈 등급

#### 거래처 통계·분석
| 기능 | 설명 |
|------|------|
| 주문 이력 대시보드 | 거래처별 주문 이력/추이 |
| 매출 순위 (파레토) | 상위 20% 거래처 매출 기여도 |
| 활동성 등급 | AI 기반 분류 (활성→주의→휴면→이탈) |
| 이탈 예측 | 주문 패턴 이상 감지 → 알림 |
| 결제 패턴 | 평균 결제일, 연체 경향 |
| 신규 vs 기존 비율 | 유입률 추이 모니터링 |

#### 매출·품목 분석
| 기능 | 설명 |
|------|------|
| 기간별 매출 비교 | 전기 대비 증감률 자동 산출 |
| 채널별 매출 비중 | 스마트스토어/자사몰/카톡B2B |
| 품목별 순위 | 기여도/마진 분리 분석 |
| 시간대별 패턴 | 주문 피크 시간 → 인력 배치 |
| 목표 달성률 | 월별 목표 대비 실적 트래킹 |
| 시즌성 분석 | 계절별 패턴 → 재고 사전 확보 |

#### 자동 보고서
| 보고서 | 주기 | 내용 | 검수 |
|--------|------|------|------|
| 일간 매출 요약 | 매일 18시 | 매출/주문/입금/특이사항 | 자동 발송 (알림) |
| 주간 채권 현황 | 매주 목요일 | 미수금 aging, 연체, 독촉 대상 | 자동 발송 (알림) |
| 월간 손익 보고서 | 매월 1일 | 매출/매입/마진/전월 대비 | **초안 생성 → 관리자 확인 후 배포** |
| 거래처 랭킹 | 월간 | 매출 TOP20, 신규, 이탈 위험 | 자동 발송 (알림) |

## 5. Phase 3: 사내 AI 챗봇 (지식 플랫폼)

### 5.1 개요
CNC코리아의 사내 정보를 학습한 AI 챗봇입니다.
**모든 업무를 챗봇으로 처리하는 것이 아니라**, 용도에 맞는 도구를 구분합니다.

### 5.2 AI vs 규칙 기반 구분

| 영역 | 적합한 도구 | 이유 |
|------|------------|------|
| 재고 수량 조회 | **대시보드 (Phase 2)** | 정형 데이터 → 검색/필터가 더 빠름 |
| 매출 추이 확인 | **대시보드 (Phase 2)** | 차트 시각화가 더 효과적 |
| 복합 질의 ("A사 최근 3개월 주문 + 미수금 + 계약 조건") | **AI 챗봇** | 여러 소스를 결합한 자연어 응답 필요 |
| 업무 매뉴얼 검색 | **AI 챗봇** | 문서 내 맥락 이해 + 요약 필요 |
| 거래처 담당자 연락처 | **검색 포털** | 단순 조회 → DB 검색이 정확 |
| 견적 단가 추천 | **AI 챗봇** | 과거 패턴 분석 + 추천 로직 |
| 급여 조회 | **접근 불가** | 고위험 — AI 챗봇에서 제외 |
| 계약서 원문 조회 | **권한 제한 AI** | 관리자 권한 필요, 열람 로그 기록 |

### 5.3 고위험 업무 제한 원칙

| 등급 | 데이터 유형 | AI 챗봇 접근 | 제한 사유 |
|------|------------|-------------|-----------|
| **Level 1** (공개) | 제품 정보, 일반 매뉴얼 | 전체 접근 | — |
| **Level 2** (내부) | 매출/재고/거래처 | 로그인 사용자만 | 영업 비밀 |
| **Level 3** (기밀) | 계약서, 단가표 | 관리자 권한 + 열람 로그 | 거래 조건 |
| **Level 4** (민감) | 급여, 인사 정보 | **접근 차단** | 개인정보보호법 |

### 5.4 학습 데이터 범위

| 데이터 소스 | 접근 방식 | 갱신 주기 |
|------------|-----------|-----------|
| 이카운트 ERP | API 실시간 호출 (Tool Use) | 실시간 |
| 업무 매뉴얼 | 문서 벡터화 (RAG) | 문서 업데이트 시 |
| 계약서 | 문서 벡터화 (RAG) + 권한 제어 | 계약 변경 시 |
| 업체 정보 | DB 조회 | 정보 변경 시 |
| 업무 노하우 | 대화 이력 학습 (익명화) | 지속 축적 |

### 5.5 활용 시나리오

#### 시나리오 1: 영업 담당자
```
직원: "홍길동 상사 최근 3개월 주문 내역 보여줘"
AI:   홍길동 상사 (C001)
      - 4월: A형 500개, B형 200개 (1,200만원)
      - 3월: A형 300개 (600만원)
      - 2월: A형 400개, C형 100개 (950만원)
      ※ 전분기 대비 25% 증가 | 미수금 230만원 (결제예정 4/30)
```

#### 시나리오 2: 물류 담당자
```
직원: "오늘 출고할 주문 리스트"
AI:   오늘 출고 예정 8건:
      1. 홍길동 상사 — A형 500개 (CJ대한통운)
      2. 김영수 — B형 50개 (한진택배)
      ※ A형 재고: 2,300개 (출고 후 1,800개)
      ※ 재고 부족 예상일: 5/15
```

#### 시나리오 3: 경영진
```
대표: "이번 주 매출 요약"
AI:   금주 매출: 4,850만원 (전주 대비 +12%)
      주문 47건 | 스마트스토어 45% / 카톡B2B 35% / 자사몰 20%
      연간 목표 달성률: 35% (현 추세 시 12.8억 예상)
```

### 5.6 기술 구성

| 기술 | 역할 |
|------|------|
| **RAG** | 사내 문서 벡터화 → 질문 관련 문서 검색 → 정확한 답변 |
| **Tool Use** | AI가 이카운트 API를 직접 호출하여 실시간 데이터 조회 |
| **MCP 서버** | Claude Desktop/웹에서 이카운트 도구를 자연어로 사용 |
| **Vector DB** | 문서 임베딩 저장 (pgvector 또는 Chroma) |

## 6. Phase 4: 물류센터 확장

### 6.1 개요
CNC코리아 물류센터 운영을 AI 플랫폼에 통합하여, 주문→출고→배송→정산 End-to-End 자동화를 구현합니다.

### 6.2 핵심 기능

#### 입고 관리
| 기능 | 설명 |
|------|------|
| 바코드/QR 스캔 | 입고 수량 자동 입력 → 이카운트 재고 반영 |
| AI 이미지 검수 | 불량/파손 자동 감지 |
| 납품 품질 분석 | 공급업체별 입고 이력 → 품질 트렌드 |

#### 재고·창고 관리
| 기능 | 설명 |
|------|------|
| 로케이션 관리 | 창고 내 위치 추적 |
| AI 안전재고 예측 | 수요 패턴 학습 → 자동 발주점 조정 |
| ABC 분석 | 품목별 회전율 기반 자동 등급화 |
| 재고 실사 | 태블릿 스캔 → 차이 자동 보고 |

#### 출고·배송 관리
| 기능 | 설명 | 검수 |
|------|------|------|
| AI 피킹 최적화 | 동선 최적화 피킹 리스트 자동 생성 | 리스트 확인 후 실행 |
| 택배사 연동 | CJ/한진/롯데/우체국 API | 자동 |
| 송장 동기화 | 이카운트 + 카페24 동시 업데이트 | 자동 (로그 기록) |
| 배송 추적 | 실시간 상태 → 완료 시 자동 정산 | 정산 건 확인 |

### 6.3 물류 분석 대시보드
일일 입출고량, 창고 가동률, 피킹 효율, 배송 리드타임, 반품·클레임률을 실시간 모니터링합니다.

## 7. Phase별 의사결정 게이트

### 7.1 Phase 1 → Phase 2 진입 조건

| 기준 | 목표 | 측정 방법 |
|------|------|-----------|
| 핵심 워크플로우 가동률 | 주 5일 이상 정상 사용 | 사용 로그 |
| 업무 시간 절감 | 기준선 대비 50% 이상 절감 | 전후 측정 비교 |
| AI 파싱 정확도 | 95% 이상 (필드 단위) | 샘플 30건 재측정 |
| 사용자 만족도 | 5점 만점 3.5 이상 | 설문 (사용자 전원) |

**중단 기준:** 파싱 정확도 80% 미만 또는 사용자 만족도 2.5 미만 → Phase 2 보류, Phase 1 개선에 집중

### 7.2 Phase 2 → Phase 3 진입 조건

| 기준 | 목표 | 측정 방법 |
|------|------|-----------|
| 대시보드 일간 접속률 | 담당자 80% 이상 | 접속 로그 |
| 보고서 활용도 | 월 3회 이상 열람/다운로드 | 다운로드 로그 |
| 데이터 정확도 | 이카운트 원본 대비 99% 일치 | 월간 대사 |
| 사내 문서 디지털화 | 핵심 매뉴얼 20건 이상 준비 | 문서 카운트 |

**중단 기준:** 대시보드 접속률 30% 미만 → 사용자 필요 재분석, Phase 3 보류

### 7.3 Phase 3 → Phase 4 진입 조건

| 기준 | 목표 | 측정 방법 |
|------|------|-----------|
| 챗봇 일간 질의 수 | 10건 이상 | 대화 로그 |
| 답변 정확도 | 90% 이상 (사용자 피드백 기준) | 👍/👎 피드백 |
| 물류센터 운영 규모 | 일 출고 50건 이상 | 현장 확인 |
| 물류 시스템 필요성 | 현업 요청 또는 오배송률 1% 이상 | 현장 데이터 |

**중단 기준:** 물류센터 규모가 일 출고 20건 미만 → Phase 4 불필요, 현행 유지

## 9. 대안 비교

### 9.1 접근 방식 비교

| 구분 | 본 제안 (AI 자동화) | RPA (UiPath 등) | BI 도구 (Tableau 등) | 이카운트 기본 기능 |
|------|-------------------|-----------------|---------------------|-----------------|
| **초기 비용** | 800만원~ | 2,000만원~ | 1,500만원~ | 0원 |
| **월 운영비** | 5만원~ | 50만원~ (라이선스) | 30만원~ (라이선스) | 0원 |
| **구축 기간** | 4~6주 | 8~12주 | 6~8주 | — |
| **OCR/AI 파싱** | 가능 (Claude) | 제한적 | 불가 | 불가 |
| **비정형 문서** | 강점 (LLM) | 약점 | 불가 | 불가 |
| **대시보드** | Phase 2에서 제공 | 별도 구축 | 강점 | 기본 제공 |
| **확장성** | 챗봇/물류 확장 | 시나리오 추가 | 대시보드 추가 | 제한적 |
| **유지보수** | 코드 레벨 | 시나리오 수정 | 설정 변경 | 이카운트 의존 |
| **이카운트 연동** | Open API 직접 | 화면 조작 (불안정) | API 또는 DB | 네이티브 |

### 9.2 왜 AI 자동화인가

1. **비정형 문서 처리**: 카톡 주문, PDF 인보이스, 영수증 — RPA/BI로는 처리 불가
2. **이카운트 API 직접 연동**: 화면 조작 RPA보다 안정적, 이카운트 UI 변경에 영향 없음
3. **단계적 확장**: 자동화(Phase 1) → 분석(Phase 2) → 지식(Phase 3) → 물류(Phase 4)
4. **낮은 진입 비용**: Phase 1만으로 800만원, 8개월 회수
